# 폴더 패널 기본 표시 및 새 항목 생성 흐름 개선

- **ID**: 003
- **날짜**: 2026-06-22
- **유형**: 버그 수정

## 작업 요약
폴더 목록 패널을 노트 화면에서 기본 표시 상태로 고정하고, frameless 잔여 여백으로 보이던 상단 패딩을 줄였습니다. 새 폴더 생성 직후 이름 입력 모드로 전환해 바로 수정할 수 있게 했고, 새 노트 생성 후 활성 노트 선택과 저장 전 H1 제목 연동 흐름을 보강했습니다.

## 원문 요청사항
```text
폴더 목록 패널
- 상단에 여백이 너무 많아. frameless일 때의 잔재가 남아있는 것으로 보여.
- 폴더 목록 패널을 숨기지 말고 기본적으로 보이도록 해줘. 
- 폴더 생성 시 "새 폴더"로만 생성이 되는데, 생성 후 자동으로 수정 모드로 하고 focus를 줘서 바로 수정할 수 있도록 해줘.

노트 목록 패널
- +를 누르면 새 노트가 생성되는데, 새로 만든 노트에 활성화 상태가 먹여지지 않는 것 같아.
- +를 누르면 새 노트가 생성되는데, 처음 저장이 되기 전까지는 맨 위의 h1 제목이 노트 이름과 연동이 되도록 해줘.
```

## 변경 파일 목록
- `src/app/component.nav.sidebar/view.ts`: 폴더 패널 기본 열림 처리, 새 폴더 생성 후 rename focus 전환, 새 노트 생성 직후 activeNoteKey 우선 반영, 저장 전 제목 변경 이벤트 수신을 추가.
- `src/app/component.nav.sidebar/view.pug`: 폴더 패널 상단 로고 영역 하단 여백 축소.
- `src/app/layout.sidebar/view.ts`: 노트 화면에서 workspace panel 상태를 기본 열림으로 유지하고 settings route에서만 닫히도록 조정.
- `src/app/page.notes/view.ts`: draft 노트의 첫 H1 제목을 첫 수동 저장 전까지 노트 제목으로 동기화하고, 직접 제목 수정 시 자동 연동을 멈추도록 처리.
